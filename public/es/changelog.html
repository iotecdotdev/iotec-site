Changelog

Última actualización: 2026-05-17

Qué hay nuevo en cada producto iotec. Los clientes actuales reciben correo al lanzarse una nueva versión. Las actualizaciones son siempre gratis — sin re-compra para iotec Core, sin interrupción del servicio para suscriptores de iotec GEX.

iotec GEX

v0.3.1 — 2026-05-17

  • UX pulido. El campo "API Token" en las propiedades del indicador ya no está visible — tu license key se usa automáticamente para el servicio GEX. Un solo secreto, una sola vez.
  • Estado del indicador en el título del chart. El nombre del indicador ahora muestra estados claros: (Loading...), (Market closed), (API offline — retrying), (HTTP error — check token). Se acabó adivinar qué pasa.
  • El título del chart ahora se refresca correctamente con mercado cerrado (antes se quedaba pegado en "Loading...").

v0.3.0 — 2026-05-17

  • Un solo secreto para todo. Tu license key ahora sirve también como API token internamente. Sin tokens separados — solo la license key.
  • Cambio a modelo BYO (Bring Your Own) Schwab: pago único, tú aportas tu propia app gratuita del Schwab Developer Portal.

v0.2.0 — 2026-05-17

  • Interno: extracción de HTTP, parsing JSON y cifrado de credenciales al DLL ofuscado. Mejor protección del flujo de datos.

v0.1.0 — 2026-05-16

  • Lanzamiento inicial. Niveles de Gamma Exposure en tiempo real en charts de NinjaTrader 8, alimentados desde un cálculo server-side con data de opciones en vivo.
  • Key levels (zero gamma, call wall, put wall), barras GEX históricas, histograma de convexidad, conversión de futuros para ES/NQ/GC.

iotec Core

v0.5.0 — 2026-05-16

  • Estadísticas de big-trade (media, desviación estándar) calculadas en C++ nativo para mayor performance.

v0.4.4 — 2026-05-16

  • Detección de revocación de licencia más rápida (heartbeat cada 15 minutos). Si tu licencia se revoca, el indicador lo detecta en minutos y se detiene limpiamente con un mensaje claro.

v0.4.0 — 2026-05-16

  • Interno: protocolo de handshake entre componentes managed y native. Mejor resistencia a manipulación.

v0.3.0 — 2026-05-16

  • Detección de imbalance (diagonal bid/ask) y cálculo de radio de big-trade portados a C++ nativo, mejor velocidad y protección.

v0.1.0 — 2026-05-15

  • Lanzamiento inicial. Toolkit completo de Order Flow en NT8: footprint con cascada de clasificación bid/ask, volumen profile, delta, marks de big-trade. Sin requerir suscripción a NinjaTrader OrderFlow+.

Convenciones

Las versiones siguen semantic versioning: MAJOR.MINOR.PATCH. PATCH es un bug fix, MINOR es feature o mejora, MAJOR indica breaking changes (muy raros y siempre anunciados con antelación por correo).
